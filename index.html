Overview

If you run latency-sensitive code, you may find SLJtest useful. It's a tool for measuring and visualizing system latency jitter. A rough definition of system latency is the time delay between when your code is ready to run and when it actually gets a CPU core and starts running. System latency can be caused by things happening inside the CPU chip, on the mainboard, in memory, in the BIOS, and in the operating system. There's a more formal definition of system latency in the doc. If we measure system latency a million times, we will see some variance (or "jitter") between measurements. Running SLJtest produces jitter statistics for minimum, average, maximum, and standard deviation that can be helpful in benchmarking and tuning. It also keeps a histogram that is useful for visualizing system latency jitter and diagnosing possible causes. SLJtest requires an x86 processor for the RDTSC instruction. It is known to run on Linux, Solaris, Mac OS, FreeBSD, and Windows.
